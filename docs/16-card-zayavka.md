# 16 · Карточка заявки

Детальный экран заявки/процедуры — открывается кликом из любой строки (Комплектация, В закупке, В сопровождении, Дашборд). Здесь ведётся бо́льшая часть данных. Опирается на `01-domain-model.md`, `02-statuses.md`, `03-roles-permissions.md`.

Карточка имеет **два режима**: заявка-родитель (пока в «Комплектации») и процедура (в «В закупке» / «В сопровождении»). Состав действий — по роли и блоку.

---

## 1. Маршрут заявки (общий индикатор)

**3 шага:** Создана → В закупке → В сопровождении. Сбоку — **отметка оплат** (УПД параллельно). Отдельного шага «Закрыта» нет.
Прогресс позиций: **«Поставлено X из Y»**.

---

## 2. Режим А — карточка заявки-родителя (в «Комплектации»)

**Шапка:** `Т-67`, наименование, тип МТР, дата загрузки, желаемый срок, составитель.

**Позиции заявки** — таблица с **ведением**: добавить / изменить / удалить / **копировать** строку + **вставка из Excel**. Колонки: Наименование, Кол-во, Ед. изм., ГОСТ/ТУ, Шифр документации (**без цены**).

**Действия (по роли):** Комплектация — редактировать (пока `Ожидает закупки`), дублировать, отменить/вернуть; Закупки — «Взять в работу».

**Комментарии** (лента + ввод).

---

## 3. Режим Б1 — карточка процедуры в «В закупке»

**Шапка:** `Т-67` + **переключатель сестёр**; № заявки + № процедуры; поставщик, тип МТР; **ФИО закупщика** (поле ввода); сумма (Σ позиций).
Поля: даты публикации, рабочий статус.

**Позиции процедуры** — **ведение**: добавить / изменить / удалить / **копировать** + **Excel-вставка** + **цена с НДС**.

**Действия (Закупки):** разбить по поставщикам, выбрать поставщика, выставить рабочий статус, отменить процедуру, передать в сопровождение (из `На сделку`).

**Комментарии**, **«История»**.

---

## 4. Режим Б2 — карточка процедуры в «В сопровождении»

**Шапка:** `Т-67` + **переключатель сестёр**; № заявки + № процедуры; поставщик, **договор**, тип МТР; **ФИО договорника** (поле ввода); **сумма договора**.
Поля: статус сделки (справочник), статус поставки (вручную), Срок ДД, План, Факт.

**Поставки** — каждая поставка:
- таблица позиций;
- **документы** — кнопки **ТТН / М-15 / УПД / Серт**: клик переключает статус **красный с крестиком ↔ зелёный с галкой**;
- № УПД, статус оплаты.
- Действия с поставкой: **Создать поставку** (выбрать позиции из «ожидают отгрузки»), **Расформировать поставку** (позиции возвращаются в «ожидают отгрузки»; доступно **только пока поставка не получена — `transit`**), отметить получение (`transit → done`).

**«Ожидают отгрузки»** — позиции без поставки (кнопка «Создать поставку»).

**Ввод № УПД** → запись появляется в «Оплаты» (`Ожидает оплаты`).

**Комментарии**, **«История»**.

---

## 5. Переключатель «сестёр»

В шапке режима Б — переключение между процедурами одной заявки (Торга): `№проц.1 ⇄ №проц.2 ⇄ …`. Процедуры движутся по статусам независимо.

---

## 6. Комментарии и история

- **Комментарии:** лента + поле ввода; автор = текущий вошедший пользователь; привязка к заявке-родителю или процедуре.
- **«История»:** аудит действий по заявке/процедуре (кто, что, когда).

---

## 7. Действия по роли (свод)

Полный перечень переходов — `02-statuses.md` §7; права — `03-roles-permissions.md`. На карточке доступны только действия, разрешённые роли текущего пользователя для текущего блока; остальным — просмотр.

---

## 8. Открытые вопросы

1. **Состав «Истории»** (какие события логируются) — детализируем на Этапе 3.
2. ✅ **Расформирование поставки** — только пока она **не получена** (`transit`); для полученной (`done`) недоступно.

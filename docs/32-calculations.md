# 32 · Расчётные правила

Точные формулы производных величин (нигде не хранятся — считаются на лету). Поля — из `30-db-schema.md`. Суммы — в копейках; отображение ₽ = `amount / 100`.

---

## 1. Суммы

- **Сумма позиции** = `qty * price`.
- **Сумма процедуры** (по умолчанию для `contract_sum`) = Σ сумм позиций процедуры. Может быть **переопределена вручную**.
- **Сумма УПД** (`amount`): для `delivery` — Σ позиций поставки; для `manual` — ручной ввод (опц. Σ `upd_position`).

---

## 2. Прогресс «Поставлено X из Y» (процедура)

- **Y** = число позиций процедуры (`procedure_position`).
- **X** = число позиций, чья поставка получена (`delivery.status = 'done'`).
- **Прогресс %** = `X / Y * 100` (если `Y = 0` → 0).

---

## 3. Просрочка (производный признак)

«Сегодня» — по часовому поясу **Москвы (Europe/Moscow)**. Дедлайн поставки — `procedure.srok_dd`.

- **Поставка просрочена:** не `done` и `srok_dd < сегодня`.
- **Поставка с задержкой:** `done` и `date > srok_dd`.
- **Процедура просрочена:** есть хотя бы одна просроченная поставка **или** просрочка по `srok_dd` при незавершённой поставке.
- **Оплата (УПД) просрочена:** `pay_status = 'await'` и `srok < сегодня`.

---

## 4. «Просроч., %» (колонка в «В сопровождении»)

Доля позиций с нарушением срока:
```
% = (позиции в поставках со статусом «просрочена» ИЛИ «с задержкой») / всего позиций процедуры * 100
```
Цвет: `0` — зелёный · `>0` — оранжевый · `≥50` — красный.

---

## 5. Агрегат документов (4 квадрата ТТН/М-15/УПД/Серт)

Документ считается **полученным**, если соответствующий флаг (`doc_ttn` / `doc_m15` / `doc_upd` / `doc_sert`) установлен **во всех** поставках процедуры. Зелёный = есть во всех, красный = нет хотя бы в одной. Если поставок нет — все красные.

---

## 6. Показатели дашборда (6)

**Общее правило:** во всех показателях **`Отменена` исключается**; завершённые процедуры (`Поставлено` + все УПД `Оплачено`) **не входят в операционные счётчики** «В закупке»/«В сопровождении», но **учитываются в финансовых итогах** (§7).

| Показатель | Формула |
|---|---|
| В закупке | `count(block='zakupka' AND status_zakup ≠ 'Отменена')` |
| В сопровождении | `count(block='soprovozhdenie' AND не завершена AND status_postavki ≠ 'Отменена')` + Σ `contract_sum` по ним |
| Поставки в срок | `поставки в срок / всего поставок * 100` (см. §9) |
| Просрочено | `count(просроченных процедур, кроме 'Отменена')` + Σ их сумм |
| УПД в оплате | `count(upd_payment where pay_status='await')` + Σ `amount` |
| УПД просрочено | `count(просроченных УПД)` + Σ `amount` |

---

## 7. Сводка «Оплаты»

**Показатели:**
- **Оплачено** = Σ `amount` где `paid`.
- **К оплате** = Σ `amount` где `await`.
- **Просрочено** = Σ `amount` просроченных УПД.
- **Сумма в работе** = Оплачено + К оплате (все активные УПД).

**Полоса распределения** (доли от общей суммы законтрактованного):
- **оплачено** = Σ `amount` `paid`;
- **выставлено-ждёт** = Σ `amount` `await`;
- **поставлено-без-УПД** = Σ сумм доставленных позиций, не вошедших ни в одну УПД;
- **законтрактовано-не-поставлено** = Σ сумм позиций недоставленных процедур.

> Финансовые итоги «Оплаты» учитывают **все** УПД (в т.ч. по завершённым процедурам). Исключаются только УПД, относящиеся к **отменённым** процедурам.

---

## 8. Отчёты

1. **Время на этапе / зависания:** `дней на этапе = сегодня − procedure.block_entered_at`; флаг «зависло», если **≥ 3 дней**.
2. **Суммы по этапам и поставщикам:** Σ `contract_sum`, сгруппировано по `block` и по `supplier`.
3. **Просрочки:** список просроченных поставок (§3) и просроченных УПД.
4. **Сводка по составителям/отделам:** `count` заявок и Σ сумм, сгруппировано по `sostavitel` / `dept`.

> В суммовых отчётах **`Отменена` исключается**; завершённые процедуры **учитываются** (финансовая полнота). Признак активности — `02-statuses.md` §7.1.

---

## 9. Решения

1. ✅ Добавлено служебное поле `procedure.block_entered_at` (обновляется при переходе; **пользователям не показывается**).
2. ✅ «Поставки в срок» — знаменатель = **все поставки**.
3. ✅ Порог «зависания» — **3 и более дней** на этапе.
4. ✅ «Сегодня» и все расчёты — по часовому поясу **Москвы**.
5. ✅ `Отменена` исключается из всех показателей; завершённые — вне операционных счётчиков, но в финансовых итогах (§6–§8).

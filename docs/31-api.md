# 31 · API (FastAPI)

REST-эндпоинты под действия со страниц. Тела запросов/ответов соответствуют полям сущностей из `30-db-schema.md` / `01-domain-model.md`. Права — `03-roles-permissions.md`.

## Общее

- **Авторизация:** сессия в httpOnly-cookie (логин/пароль, тайм-аут простоя 120 мин, «Запомнить меня» — продлённая cookie). 2FA нет.
- **В путях — числовой `id`**; человекочитаемые коды (`Т-67`, `№1488`, `АП…`) — для поиска/отображения.
- **Суммы** в теле — целые копейки.
- **Коды ошибок:** `401` не авторизован · `403` нет прав · `404` не найдено · `409` конфликт (напр. дубль `code`) · `422` ошибка валидации.
- Каждый изменяющий запрос пишет запись в `audit_log`.
- **Пагинация** списков: `?page=&page_size=` (по умолчанию `page_size=50`); ответ включает `total`.
- **Активные по умолчанию:** GET-списки рабочих страниц возвращают только активные записи; `?include_archived=1` добавляет завершённые/отменённые (`02-statuses.md` §7.1). Реестр «Оплаты» по умолчанию отдаёт все УПД.

## 1. Авторизация и пользователи

| Метод | Путь | Кто | Назначение |
|---|---|---|---|
| POST | `/auth/login` | все | `{email, password, remember_me}` → cookie |
| POST | `/auth/logout` | авторизованные | завершить сессию |
| GET | `/auth/me` | авторизованные | текущий пользователь и права |
| POST | `/auth/change-password` | авторизованные | `{current, new}` |
| GET | `/users` | Админ | список |
| POST | `/users` | Админ | создать `{email, full_name, account_type, department, is_curator, global_role, password}` |
| PATCH | `/users/{id}` | Админ | изменить / заблокировать (`is_active`) |
| POST | `/users/{id}/reset-password` | Админ | сброс пароля (ставит `must_change_password`) |

> **Смена пароля при первом входе:** если `must_change_password=1` (предзаполненный Админ, новый пользователь, после сброса), `/auth/me` это возвращает; до успешного `/auth/change-password` остальные изменяющие запросы отклоняются (`403`). См. `04-auth.md`.

## 8. Проверка прав (на каждом изменяющем эндпоинте)

- Действия отдела — только своему этапу (Комплектация/Закупки/Сопровождение).
- Куратор отдела — полный доступ к своему блоку; Админ — ко всему; Руководитель — только `GET`.
- `/reports/*` — Руководитель/Админ/Куратор (Куратор видит все данные).
- `/users/*`, `/dict/*` (запись) — только Админ.

## 9. Точки расширения (заложены, реализуются позже — `34-extensions.md`)

В MVP **не реализуются**; перечислены, чтобы дизайн API их не блокировал. Интеграционные эндпоинты аутентифицируются **сервисным токеном**, а не пользовательской cookie.

| Метод | Путь | Назначение |
|---|---|---|
| POST | `/integrations/1c/payments` | внешнее обновление статуса оплаты УПД (сопоставление по `ext_id` / № УПД) |
| POST/GET/DELETE | `/{entity}/{id}/attachments`, `/attachments/{id}` | вложения-файлы (multipart) |
| — | (уведомления) | фоновый воркер + outbox; HTTP-эндпоинт не обязателен |

## 10. Решения / открытые вопросы

1. ✅ **Авторизация — сессия в httpOnly-cookie** (логин/пароль, 120 мин простоя, «Запомнить меня»).
2. ✅ **Расформирование поставки** — только пока `transit` (для `done` → `409`).
3. ✅ **Пагинация** `page_size=50`; **активные по умолчанию** (`?include_archived`).
4. ✅ **Глобальный поиск** — `/search` (§6.1).
5. ✅ **Смена пароля при первом входе** — `must_change_password` (§1).
6. Полные JSON-схемы тел запросов/ответов — генерируются из моделей FastAPI (Pydantic) по полям из `30-db-schema.md`; отдельной детализацией займёмся при реализации.

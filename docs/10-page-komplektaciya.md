# 10 · Страница «Комплектация»

Первый блок маршрута: заявки от комплектовщиков, ожидающие закупки. Опирается на `01-domain-model.md` (Заявка-родитель, Запрошенная позиция), `02-statuses.md` §2, `03-roles-permissions.md`.

---

## 1. Назначение и доступ

Единое окно для заявок-родителей (`Т-67`), которые завела Комплектация и которые ждут, когда Закупки возьмут их в работу.

| Роль | Доступ на странице |
|---|---|
| Комплектация (сотрудник) | Создавать, редактировать, дублировать, отменять/возвращать заявки |
| Закупки (сотрудник) | «Взять в работу» |
| Куратор Комплектации, Админ | Полный доступ к блоку (все действия + правка/удаление любых записей) |
| Руководитель | Только просмотр |
| Остальные | Просмотр (единое окно) |

---

## 2. Макет

- Заголовок блока **«Ожидают закупки»** + счётчик заявок.
- Кнопки в шапке: **«+ Заявка»** (создать), **«Экспорт»** (в Excel).
- Панель **фильтра/сортировки/поиска**.
- **Таблица** заявок.

---

## 3. Таблица

| Колонка | Источник | Формат |
|---|---|---|
| # | порядковый | число |
| Наименование заявки | `code` + `title` | `Т-67` тегом + название |
| Тип МТР | `mtr` | текст |
| Срок поставки | `srok` | дата |
| Дата загрузки | `zagruzka` | дата |
| Составитель | `sostavitel` | ФИО |
| Позиции | кол-во запрошенных позиций | число |
| Статус | `status` | чип: `Ожидает закупки` / `Отменена` |

- **Сортировка:** по наименованию, коду, типу МТР, дате загрузки.
- **Фильтр:** по статусу. **По умолчанию — только активные** (`Ожидает закупки`, ещё не взятые в работу); `Отменена` — под переключателем «Показать отменённые» (`02-statuses.md` §7.1).
- **Поиск:** по коду, наименованию и составителю (плюс глобальный поиск в шапке).
- **Клик по строке** → карточка заявки (`16-card-zayavka.md`).

---

## 4. Создание заявки («+ Заявка»)

**Поля шапки (вручную):**
- Код `Т-67` (уникальный), Наименование, Тип МТР, Желаемый срок поставки.

**Проставляется автоматически:**
- Составитель = текущий вошедший пользователь; Дата загрузки = дата создания.

**Таблица позиций** — колонки: **Наименование, Кол-во, Ед. изм., ГОСТ/ТУ, Шифр документации**. **Без цены** (цену проставит закупщик).
- Ввод **вручную** (построчно) **и массовая вставка из Excel**: скопировать диапазон в Excel/Sheets и вставить — строки заполняются автоматически (буфер обмена в формате TSV). *Реализация — компонент Excel-подобной таблицы со вставкой; деталь — Этап 3.*

При сохранении статус = `Ожидает закупки`.

---

## 5. Действия

| Действие | Кто | Результат |
|---|---|---|
| + Заявка | Комплектация | Новая заявка, статус `Ожидает закупки` |
| Редактировать | Комплектация | Правка полей и позиций — **только пока `Ожидает закупки`** |
| Дублировать заявку | Комплектация | Копия заявки (новый код — см. §9) |
| Отменить заявку | Комплектация | `Ожидает закупки → Отменена` (остаётся в блоке) |
| Вернуть из отмены | Комплектация | `Отменена → Ожидает закупки` (обратимо) |
| Взять в работу | Закупки | **Из строки таблицы или из карточки**; заявка уходит в «В закупке», статус `Новая` |
| Экспорт | роли с доступом | Выгрузка таблицы в Excel |

Куратор Комплектации и Админ — все действия блока; Руководитель — без действий.

---

## 6. Навигация

- Строка → карточка заявки.
- «Взять в работу» → заявка исчезает из «Комплектации» и появляется в «В закупке».

---

## 7. Состояния

- **Пусто:** нет заявок «Ожидают закупки» — заглушка.
- **Нет прав:** для ролей без права действий (напр. Руководитель) кнопки скрыты/недоступны, доступен просмотр.

---

## 8. Бизнес-правила

- Редактирование возможно **только при статусе `Ожидает закупки`** (после «Взять в работу» — нельзя).
- Отменённые заявки **остаются в блоке** (для отчётности); отмена **обратима**.
- Позиции на этом этапе — **без цены**.
- Код `Т-67` уникален, вводится вручную.
- **«Ожидают закупки» = заявки без созданных процедур.** После «Взять в работу» (появляется процедура) заявка уходит из блока в «В закупке».

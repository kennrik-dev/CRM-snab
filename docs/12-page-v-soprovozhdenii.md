# 12 · Страница «В сопровождении»

Процедуры после закупки: договор, поставки (частичные), документы, УПД. Опирается на `01-domain-model.md` (Процедура, Поставка, УПД), `02-statuses.md` §4–5, `03-roles-permissions.md`. Детальное ведение — в карточке заявки (`16-card-zayavka.md`).

---

## 1. Назначение и доступ

Сопровождение ведёт переданную из закупки процедуру: вводит договор, создаёт поставки, отмечает документы, выставляет УПД, ведёт статусы.

| Роль | Доступ |
|---|---|
| Сопровождение (сотрудник) | Все действия сопровождения (см. §5) |
| Куратор Сопровождения, Админ | Полный доступ к блоку **и к «Оплаты»** (+ правка/удаление любых записей) |
| Руководитель | Только просмотр |
| Остальные | Просмотр (единое окно) |

**Видимость:** по умолчанию — **только активные** (не завершённые и ≠ `Отменена`); завершённые/отменённые — под переключателем (`02-statuses.md` §7.1).

## 4. Что ведётся в карточке (детально — `16-card-zayavka.md`)

- **Договор:** Сопровождение вводит № и дату договора + сумму (авто Σ позиций с НДС, **правится вручную**).
- **Поставки (частичные):** создаются в карточке — выбор позиций из «ожидают отгрузки» (**поставка не пустая**); у каждой свой статус (`transit → done`), даты (факт/ETA), документы, № УПД.
- **Документы:** ТТН / М-15 / УПД / Серт — отмечаются по каждой поставке; в таблице страницы — агрегат «есть во всех поставках».
- **№ УПД:** ввод создаёт запись в «Оплаты» со статусом `Ожидает оплаты`.

| Действие | Кто | Где | Результат |

| Ввести договор + сумму | Сопровождение | карточка | № и дата договора, сумма |

| Создать поставку | Сопровождение | карточка | поставка с позициями (`transit`) |
| Отметить получение | Сопровождение | карточка | `transit → done` |
| Отметить документы | Сопровождение | карточка | ТТН/М-15/УПД/Серт |

## 7. Состояния

- **Пусто:** нет процедур в сопровождении.
- **Нет прав:** Руководитель и сторонние роли — только просмотр.

---

## 8. Бизнес-правила

- **Договор и сумму** вводит **Сопровождение** (не Закупки). Сумма = авто Σ позиций (с НДС), ручная правка.
- **Поставка не пустая** (≥1 позиция из «ожидают отгрузки»).
- **Документы** — полный набор ТТН/М-15/УПД/Серт; в таблице — «есть во всех поставках».
- **Статус поставки** — вручную; **просрочка** — производная (Срок ДД прошёл, а статус ≠ `Поставлено`, либо Факт позже срока).
- **Завершённость** = `Поставлено` + все УПД `Оплачено` (отдельного статуса «Закрыта» нет).
- Ввод **№ УПД** → запись появляется в «Оплаты».

---

## 9. Решения / уточнения

1. ✅ Поставки/отгрузки создаются **в карточке**; кнопка **«+ Отгрузка»** на странице — **заглушка** (вернёмся, если понадобится).
2. ✅ **План** — плановая дата поставки, свободный ввод Сопровождением. **Факт** — фактическая дата: по умолчанию подставляется дата последней полученной поставки, можно переопределить вручную.

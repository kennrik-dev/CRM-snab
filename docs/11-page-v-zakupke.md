# 11 · Страница «В закупке»

Заявки, взятые Закупками в работу: торги на ЭТП, формирование процедур, выбор поставщиков, цены. Опирается на `01-domain-model.md` (Торг, Процедура, Позиция процедуры), `02-statuses.md` §3, `03-roles-permissions.md`.

---

## 1. Назначение и доступ

Закупщик ведёт заявку в закупке: заводит № заявки/№ процедуры, дробит по поставщикам, ведёт позиции и цены, проставляет рабочие статусы до «На сделку» и передаёт в сопровождение.

| Роль | Доступ |
|---|---|
| Закупки (сотрудник) | Все действия закупки (см. §5) |
| Куратор Закупок, Админ | Полный доступ к блоку (+ правка/удаление любых записей) |
| Руководитель | Только просмотр |
| Остальные | Просмотр (единое окно) |

---

## 2. Макет

- Заголовок блока **«В закупке»** + счётчик.
- Кнопки: **«+ Заявка»** (пока **заглушка** — см. §9), **«Экспорт»** (в Excel).
- Панель **фильтров / сортировки / поиска** (расширенная — см. §3).
- **Таблица** процедур.

---

## 3. Таблица

**Единица строки — процедура** (`block = zakupka`). Сразу после «Взять в работу» это одна **процедура по умолчанию** (= вся заявка), с пустыми № заявки/№ процедуры/поставщиком и статусом `Новая`.

| Колонка | Источник | Формат |
|---|---|---|
| # | порядковый | число |
| Наименование заявки | родитель `code` + `title` | `Т-67` тегом + название |
| Тип МТР | `mtr` | текст |
| № заявки | `tender.num` | текст (может быть пустым) |
| № процедуры | `proc` | текст (может быть пустым) |
| Поставщик | `supplier` | текст (`—` пока не выбран) |
| Дата загрузки | родитель `zagruzka` | дата |
| Нач. публ. | `pub_start` | дата |
| Заверш. публ. | `pub_end` | дата |
| Поз. | кол-во позиций процедуры | число |
| Статус | `status_zakup` | чип: `Новая` … `На сделку` / `Отменена` |

**Редактирование — в строке и в карточке заявки** (`16-card-zayavka.md`): № заявки, № процедуры, поставщик, даты публикации, рабочий статус. Клик по строке → карточка.

**Фильтры (по максимуму):** по рабочему статусу, по поставщику, по типу МТР, по наличию/отсутствию поставщика, по диапазону дат публикации, по дате загрузки, по составителю.
**Сортировка:** по коду `Т-67`, № заявки, № процедуры, типу МТР, поставщику, дате загрузки, датам публикации, статусу.
**Поиск:** по `Т-67`, № заявки, № процедуры, поставщику, наименованию заявки (плюс глобальный поиск в шапке).
**Видимость:** по умолчанию — **только активные** (статус ≠ `Отменена`); отменённые — под переключателем (`02-statuses.md` §7.1).

---

## 4. Дробление и ведение процедур

- После «Взять в работу» — одна **процедура по умолчанию**, статус `Новая`.
- **Разбить по поставщикам:** закупщик создаёт новую процедуру и **переносит в неё выбранные позиции** заявки (с возможностью **дробить количество**). Поставщика, № процедуры и № заявки можно указать **сразу или дополнить позже**.
- **Ведение позиций (обязательно):** закупщик **добавляет / редактирует / удаляет** позиции и проставляет **цену с НДС** — до передачи в сопровождение. Массовая **вставка из Excel** обязательна (как в «Комплектации»).
- **Выбрать поставщика:** проставляет `supplier` на процедуре.
- **Передача в сопровождение — по каждой процедуре отдельно**, когда она достигла статуса `На сделку` (процедуры одной заявки движутся независимо).

---

## 5. Действия

| Действие | Кто | Результат |
|---|---|---|
| Разбить по поставщикам | Закупки | Новая процедура + перенос выбранных позиций (с дроблением кол-ва) |
| Вести позиции (добавить/изменить/удалить, цена с НДС, вставка из Excel) | Закупки | Правка позиций процедуры |
| Ввести № заявки / № процедуры | Закупки | В строке или в карточке |
| Выбрать поставщика | Закупки | Проставляет `supplier` |
| Указать даты публикации | Закупки | `pub_start` / `pub_end` |
| Выставить рабочий статус | Закупки | `Новая` … `На сделку` |
| Отменить процедуру | Закупки | `→ Отменена` (обратимо) |
| Передать в сопровождение | Закупки | Из `На сделку` → «В сопровождении», статус поставки `Новая`. Договор вводит уже Сопровождение |
| Экспорт | роли с доступом | Выгрузка в Excel |

Куратор Закупок и Админ — все действия блока; Руководитель — без действий.

---

## 6. Навигация

- Строка → карточка заявки.
- «Передать в сопровождение» → процедура уходит в «В сопровождении».

---

## 7. Состояния

- **Пусто:** нет процедур в закупке.
- **Нет прав:** Руководитель и сторонние роли — только просмотр.

---

## 8. Бизнес-правила

- **Цену позиций (с НДС)** вводит закупщик **до** передачи в сопровождение.
- **Договор и сумму договора** вводит **Сопровождение** уже в своём блоке — не здесь.
- **Дробление:** Σ распределённого количества ≤ запрошенного по исходной позиции (`01-domain-model.md` §3); закупщик может добавлять новые позиции (`source_ref = null`).
- **Передача в сопровождение** — из статуса `На сделку`, отдельно по каждой процедуре.
- **Отмена процедуры** обратима, процедура остаётся в блоке.

---

## 9. Решения

1. ✅ **«На ЭТП»** — действие закупщика **вне системы** (отыгрывание на тендере). В системе закупщик **вручную** проставляет даты публикации и меняет статус; отдельной кнопки-автоматизации нет.
2. ⏳ **«+ Заявка» в «В закупке»** — оставляем кнопку **заглушкой**; вернёмся, если понадобится заводить заявку, минуя «Комплектацию».

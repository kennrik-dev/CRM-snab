# 04 · Авторизация и пользователи

Вход, учётные записи, пароли и сессии. Права ролей — `03-roles-permissions.md`, сущность пользователя — `01-domain-model.md` §2.8.

---

## 1. Вход

- Вход по **email + паролю**. **Самостоятельной регистрации нет** — учётные записи создаёт Админ.
- Неверный email/пароль → сообщение об ошибке (без подсказки, что именно неверно).
- Заблокированный (деактивированный) пользователь войти не может.

---

## 2. Учётная запись (User)

| Поле | Тип | Описание |
|---|---|---|
| email | string, уникальный | Логин для входа |
| password_hash | string | Хэш пароля (напр. bcrypt), не сам пароль |
| full_name | string | ФИО |
| account_type | enum | `department` (член отдела) / `global` (глобальная роль) |
| department | enum / null | Комплектация / Закупки / Сопровождение (если `department`) |
| is_curator | bool | Куратор отдела (если `department`) |
| global_role | enum / null | Админ / Руководитель (если `global`) |
| is_active | bool | Активна / заблокирована |
| created_at / created_by | datetime / FK | Когда и каким Админом заведена |

Права по этим полям — `03-roles-permissions.md`.

---

## 3. Управление пользователями — **только Админ**

**Первый Админ — предзаполнен.** При первом развёртывании в БД заводится одна учётка Админа с **дефолтным паролем**; вход возможен сразу, но система **требует сменить пароль при первом входе** (`must_change_password`). Через неё Админ создаёт остальных. Самостоятельной регистрации по-прежнему нет.

Куратор пользователями **не управляет**. Админ может:
- **Создать** пользователя: email, ФИО, тип (отдел + признак Куратор / глобальная роль), начальный пароль.
- **Изменить** ФИО, отдел/роль, признак Куратор.
- **Заблокировать / разблокировать** (деактивация).
- **Сбросить пароль**.

---

## 4. Пароли

- Админ задаёт **начальный пароль** при создании учётки.
- Пользователь **меняет свой пароль сам** (в профиле), указав текущий.
- **«Забыл пароль» → обращается к Админу**, тот сбрасывает (задаёт новый). **Email-восстановления нет.**
- **Принудительная смена при первом входе:** учётка с заданным «чужим» начальным паролем (предзаполненный Админ, созданный Админом пользователь, сброс пароля) помечается `must_change_password`; при следующем входе система требует задать новый пароль, прежде чем пустить в систему.
- Требования: **минимум 8 символов**.
- Хранение — только хэш.

---

## 5. Сессии

- После входа создаётся сессия.
- **Авто-выход по неактивности** через **120 минут** простоя.
- **«Запомнить меня»** — продлённая сессия на доверенном устройстве.
- Выход вручную (кнопка «Выйти»).
- **2FA не используется.** Защита от перебора паролей — не на начальном этапе.

---

## 6. Деактивация (вместо удаления)

Уволенного сотрудника **блокируют**, а не удаляют: учётка отключается, но **авторство комментариев и история действий сохраняются**. Поэтому жёсткое удаление пользователей не предусмотрено.

---

## 7. Решения по параметрам

- ✅ Тайм-аут неактивности — **120 минут**.
- ✅ «Запомнить меня» — **есть**.
- ✅ Защита от перебора — **не нужна на начальном этапе** (вернуться при размещении в интернете).
- ✅ **Первый Админ — предзаполнен** с дефолтным паролем; смена обязательна при первом входе (`must_change_password`).

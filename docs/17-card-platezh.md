# 17 · Карточка платежа

Детальный экран УПД — открывается из реестра «Оплаты». Опирается на `01-domain-model.md` §2.6 (УПД/Платёж), `02-statuses.md` §5, `03-roles-permissions.md`.

---

## 1. Назначение и доступ

Просмотр и ведение одной УПД, проведение оплаты.

| Роль | Доступ |
|---|---|
| Сопровождение (сотрудник) | Провести оплату, редактировать поля, переключать документы |
| Куратор Сопровождения, Админ | Полный доступ |
| Руководитель | Только просмотр |

---

## 2. Состав

**Шапка:** № УПД, статус оплаты (`Ожидает оплаты` / `Оплачено` / `Просрочена`), сумма с НДС.

**Поля (редактируются вручную):** Заявка (`Т-67`/№), Поставщик, Договор, № ЗРДС, Срок.

**Связанная поставка** — для УПД из поставки (`origin = delivery`): № поставки/ссылка.

**Документы** — кнопки-переключатели **ТТН / М-15 / УПД / Серт** (красный с крестиком ↔ зелёный с галкой). Для УПД из поставки; для ручной УПД — отсутствуют/прочерк.

**Позиции в составе УПД** — таблица: №, Наименование, Ед. изм., Кол-во, Цена за ед., Сумма (для ручной УПД — редактируемая; опционально).

**«История»** — аудит действий по УПД. **Комментариев нет.**

---

## 3. Действия

| Действие | Кто | Результат |
|---|---|---|
| Провести оплату | Сопровождение | `Ожидает оплаты → Оплачено`, фиксирует дату. **Только полная** |
| Редактировать поля | Сопровождение | Срок, ЗРДС, Договор, Поставщик, Сумма, позиции |
| Переключать документы | Сопровождение | ТТН/М-15/УПД/Серт |

Куратор Сопровождения и Админ — все действия; Руководитель — без действий.

---

## 4. Навигация, состояния, правила

- Из реестра «Оплаты» → карточка; кнопка «Назад» → реестр.
- **Оплата — только полная** (на начальном этапе).
- **Просрочка** — производная (срок прошёл, статус ≠ `Оплачено`).
- Документы — для УПД из поставки; для ручной УПД — нет.

---

## 5. История

Аудит по УПД: кто провёл оплату, правки полей, переключения документов (кто, что, когда).

---

## 6. Открытые вопросы

Нет (карточка определена полностью).

# 03 · Права доступа

Роли, видимость страниц и права действий. Доступ к данным — **на уровне отдела**. Механика входа (логин/пароль/сессии) — в `04-auth.md`.

---

## 1. Принцип «единое окно»

Все авторизованные пользователи **видят одни и те же заявки** на рабочих страницах (Дашборд, Комплектация, В закупке, В сопровождении, Оплаты). Различаются **права редактирования** — по роли. Страница «Отчёты» — с ограничением. *(основание — `../Concept design/SPECIFICATION.md` §1)*

---

## 2. Роли

Два вида ролей:

**Отделы** (в каждом несколько сотрудников, права на уровне отдела):

| Отдел | Зона действий |
|---|---|
| **Комплектация** | Заводят и ведут заявки-родители (блок «Комплектация») |
| **Закупки** | Берут заявки в работу, торги, процедуры (блок «В закупке») |
| **Сопровождение** | Договоры, поставки, документы, УПД, оплата (блоки «В сопровождении» и «Оплаты») |

Внутри отдела пользователь — **обычный сотрудник** или **Куратор**.

**Куратор** — руководитель отдела: полные права (как Админ) **в границах блока своего отдела** + отчёты. Привязан к одному отделу.

**Глобальные роли** (вне отделов):

| Роль | Кратко |
|---|---|
| **Админ** | Полный доступ ко всему + управление пользователями и справочниками |
| **Руководитель** | Просмотр всего (read-only) + отчёты |

---

## 3. Видимость страниц

✓ — страница доступна (просмотр), — — недоступна.

| Роль | Дашборд | Комплектация | В закупке | В сопровождении | Оплаты | Отчёты |
|---|:--:|:--:|:--:|:--:|:--:|:--:|
| Комплектация (сотрудник) | ✓ | ✓ | ✓ | ✓ | ✓ | — |
| Закупки (сотрудник) | ✓ | ✓ | ✓ | ✓ | ✓ | — |
| Сопровождение (сотрудник) | ✓ | ✓ | ✓ | ✓ | ✓ | — |
| Куратор (любого отдела) | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| Руководитель | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| Админ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |

Рабочие страницы видят все (единое окно). «Отчёты» — только Куратор, Руководитель, Админ.

---

## 4. Права действий (редактирование)

Конкретные действия и переходы статусов — в `02-statuses.md` §7. Кто их может выполнять:

| Роль | Права действий |
|---|---|
| **Отдел (сотрудник)** | Действия **своего** этапа (Комплектация / Закупки / Сопровождение — по `02-statuses.md` §7). На чужих страницах — только просмотр. |
| **Куратор отдела** | Все действия своего отдела **+ полная правка/удаление/исправление** любых записей в границах своего блока. Вне своего блока — просмотр. |
| **Руководитель** | **Нет действий** — только просмотр + отчёты. |
| **Админ** | **Все действия во всех блоках** + управление пользователями и справочниками. |

Границы блока для Куратора:

| Куратор отдела | Полный доступ к блокам |
|---|---|
| Комплектация | Комплектация |
| Закупки | В закупке |
| Сопровождение | В сопровождении **и** Оплаты |

Все Кураторы дополнительно имеют доступ к «Отчётам» — и в отчётах видят данные **по всем заявкам**, не ограничиваясь своим блоком.

---

## 5. Сводка «роль → возможности»

| Роль | Просмотр | Редактирование | Отчёты | Админка |
|---|:--:|---|:--:|:--:|
| Отдел (сотрудник) | всё | свой этап | — | — |
| Куратор отдела | всё | полный — свой блок | ✓ | — |
| Руководитель | всё | — | ✓ | — |
| Админ | всё | всё | ✓ | ✓ |

---

## 6. Решения

- ✅ **Единое окно:** все рабочие страницы видны всем (просмотр), правка по роли.
- ✅ **«Полный доступ» Куратора** = все действия отдела + правка/удаление/исправление любых записей в границах блока (в т.ч. внесённых другими сотрудниками отдела).
- ✅ **Сотрудники отделов не видят «Отчёты»** (только Куратор/Руководитель/Админ).
- **Модель пользователя:** пользователь — либо член отдела (обычный/Куратор), либо глобальная роль (Админ/Руководитель). Детали — `04-auth.md`.

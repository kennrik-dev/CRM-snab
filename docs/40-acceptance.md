## 3. Прототип vs целевая система (расхождения)

| Аспект | Концепт-прототип | Целевая система |
|---|---|---|
| Название | «СНАБ» | **CRM Ultima** |
| Вход | переключение роли селектом, без входа | **логин (email) + пароль**, отделы и роли |
| Отдел «Оплата» | отдельная роль | **убран** — оплату проводит Сопровождение |
| Модель | 2 уровня (родитель → дочерняя) | **3 уровня** (родитель → торг → процедура) |
| Статусы | единый `stage` (proc/supp/pay/done/late) | **блочная модель** + производная просрочка, без «Закрыта» |
| Лот | поле `lot` | **убрано** |
| Оплаты | стадия | **параллельный реестр УПД** + ручное добавление |
| Позиции | name/qty/unit/price | + **ГОСТ/ТУ, шифр документации**, Excel-вставка, ведение/копирование |
| Новое | — | ЗРДС, срок УПД, ФИО закупщика/договорника, расформирование поставок |

## 4. Отложенные и открытые пункты

**На подтверждение:** нет.

**Закрыто доп. проходом (полнота к реализации):**
- Расформирование поставки — только пока `transit`.
- Первый Админ — предзаполнен, смена пароля при первом входе (`04` §3–4, `30`).
- Seed справочников статусов; `Новая`/`Отменена` — служебные (`30`, `02` §3).
- Активные по умолчанию, завершённые/отменённые — под переключателем; «Ожидают закупки» = заявки без процедур (`02` §7.1, страницы 10–13).
- `Отменена` вне всех показателей; завершённые — вне операционных счётчиков, но в финитогах (`32` §6–8).
- Конкурентность — last-write-wins (`33` §7).
- Часовой пояс — Москва (`33` §1, `32` §3).
- НДС-разбивка — отложена до интеграции (`34`).
- Предусловия переходов не блокируют; «Серт» у УПД — предупреждение (`02` §7.2, `14` §5).
- Глобальный поиск + пагинация 50 (`31`, `33` §8).

**Запланированные расширения** (заложены в архитектуру, реализуются позже — `34-extensions.md`):
- Внешний учёт оплат (1С) — авто-обновление статуса оплаты УПД.
- Вложения (файлы) к заявке / процедуре / УПД.
- Уведомления пользователей по почте / мессенджеру.

**Отложено осознанно:**
- Размещение (интернет / локальный сервер) — `33` §7.
- Кнопки-заглушки: «+ Заявка» в «В закупке», «+ Отгрузка» в «В сопровождении».
- Привязка ручной УПД к процедуре (пока свободный текст) — `13` §9.
- Детали реализации: состав экспорта, глубина «Ленты событий», полные JSON-схемы API.

## 6. Критерии приёмки (проверка реализации)

- **Маршрут заявки:** Комплектация создаёт `Т-67` → Закупки «берут в работу» → дробят на процедуры (с дроблением количества) → ведут до «На сделку» → передают в сопровождение → поставки/документы/УПД → оплата. Каждый шаг — у роли, имеющей право.
- **Иерархия:** один `Т-67` даёт ≥1 торг, торг — ≥1 процедуру (без дробления — одна по умолчанию). Процедуры движутся независимо.
- **Статусы:** соответствуют блочной модели; просрочка вычисляется; «Закрыта» нет.
- **Права:** отдел — свой этап; Куратор — свой блок целиком; Руководитель — только просмотр + отчёты; Админ — всё + пользователи.
- **Деньги:** копейки внутри, ₽ с НДС в отображении; суммы авто с ручной правкой.
- **Excel-вставка** работает в позициях (Комплектация, В закупке).
- **Оплаты:** все УПД (из поставок + ручные); просрочка по сроку; оплата полная.
- **Расчёты** (прогресс, % просрочки, агрегат документов, показатели) — по формулам `32`.

# 13 · Страница «Оплаты»

Параллельный реестр всех УПД (из поставок и добавленных вручную) и контроль оплаты. Опирается на `01-domain-model.md` §2.6 (УПД/Платёж), `02-statuses.md` §5, `03-roles-permissions.md`. Карточка платежа — `17-card-platezh.md`.

---

## 1. Назначение и доступ

Реестр платежей по УПД. **«Оплаты» — параллельный блок:** процедура остаётся в «В сопровождении», а её УПД отслеживаются здесь.

| Роль | Доступ |
|---|---|
| Сопровождение (сотрудник) | Добавлять УПД, проводить оплату, вести поля (срок, ЗРДС, договор) |
| Куратор Сопровождения, Админ | Полный доступ к блоку |
| Руководитель | Только просмотр |
| Остальные | Просмотр (единое окно) |

---

## 2. Макет

- **Сводка** (4 показателя + полоса распределения).
- Кнопки: **«+ Добавить УПД»** (ручное добавление), **«Экспорт»**.
- Панель **фильтров / сортировки / поиска** (расширенная).
- Таблица **«Реестр платежей»**.

---

## 3. Сводка

- Показатели: **Сумма в работе · Оплачено · К оплате · Просрочено**.
- **Полоса распределения:** оплачено / выставлено-ждёт / поставлено-без-УПД / законтрактовано-не-поставлено (как в концепте).

---

## 4. Реестр платежей (таблица)

| Колонка | Источник | Формат |
|---|---|---|
| УПД | `upd` | текст |
| Заявка | `request_label` | `Т-67` + № |
| Поставщик | `supplier` | текст |
| Договор | `contract` (из «В сопровождении» / ручной) | текст или **прочерк** |
| ЗРДС | `zrds` (ручной) | текст или **прочерк** |
| Поставка | № поставки | № или **прочерк** (для ручных УПД) |
| Статус | `pay_status` | чип: `Ожидает оплаты` / `Оплачено` (+ `Просрочена`) |
| Срок | `srok` (ручной ввод) | дата |
| Сумма | `amount` | ₽ (с НДС) |

- Показываем **все** УПД (любой статус) — реестр полный. Быстрый переключатель **«скрыть оплаченные»** доступен, но **по умолчанию выключен** (в отличие от других страниц, где архив скрыт по умолчанию).
- Клик по строке → **карточка платежа**.

**Фильтры (по максимуму):** по статусу оплаты, наличию просрочки, поставщику, наличию договора/ЗРДС, происхождению (из поставки / ручная), диапазону срока и суммы.
**Сортировка:** по УПД, заявке, поставщику, договору, ЗРДС, статусу, сроку, сумме.
**Поиск:** по № УПД, заявке (`Т-67`/№), поставщику, договору, ЗРДС.

---

## 5. Действия

| Действие | Кто | Результат |
|---|---|---|
| + Добавить УПД (вручную) | Сопровождение | Новая запись УПД (`origin = manual`), статус `Ожидает оплаты` — форма см. §6 |
| Провести оплату | Сопровождение | `Ожидает оплаты → Оплачено` (фиксирует дату). **Только полная оплата** |
| Вести поля (Срок, ЗРДС, Договор) | Сопровождение | Ручной ввод в строке / карточке платежа |
| Экспорт | роли с доступом | Выгрузка в Excel |

Куратор Сопровождения и Админ — все действия; Руководитель — без действий.

---

## 6. Форма «+ Добавить УПД» (ручное добавление)

Поля:
- **№ УПД**
- **Заявка** (свободный текст, напр. `Т-67 + №512142`)
- **Наименование поставщика**
- **Срок** (оплаты)
- **Сумма с НДС**
- **№ ЗРДС**
- **Позиции по УПД** — *опционально* (можно без них), таблицей: **№, Наименование, Ед. изм., Кол-во, Цена за ед., Сумма**.

После сохранения УПД появляется в реестре со статусом `Ожидает оплаты`.

---

## 7. Навигация и состояния

- Строка → карточка платежа (`17-card-platezh.md`).
- УПД из поставки появляется автоматически, когда Сопровождение ввело № УПД в карточке заявки.
- **Пусто:** нет УПД. **Нет прав:** Руководитель и сторонние роли — только просмотр.

---

## 8. Бизнес-правила

- Реестр содержит **все** УПД: из поставок (`origin = delivery`) и добавленные вручную (`origin = manual`).
- **Срок оплаты** — ручной ввод; **просрочка** — производная (срок прошёл, статус ≠ `Оплачено`).
- **Оплата — только полная** по УПД (на начальном этапе; частичная — позже).
- **Сумма** — авто Σ позиций (для `delivery`) или ручной ввод (для `manual`).
- **Договор** тянется из «В сопровождении» или вводится вручную; **ЗРДС** — ручной; нет данных → **прочерк**.
